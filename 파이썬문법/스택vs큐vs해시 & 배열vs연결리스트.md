# Stack vs Queue vs Hash Table

| 경우 | Stack | Queue | Hash Table                                                                                                                                  |
| :--- | :--- | :--- |:--------------------------------------------------------------------------------------------------------------------------------------------|
| **특정 원소 조회** | $O(N)$ | $O(N)$ | $\Omega(1)$ / $O(N)$                                                                                                                        |
| **중간에 삽입 삭제** | $O(N)$ | $O(N)$ | $\Omega(1)$ / $O(N)$                                                                                                                        |
| **데이터 추가** | $O(1)$ | $O(1)$ | $\Omega(1)$ / $O(N)$                                                                                                                        |
| **정리** | 가장 최근에 들어온 순서대로 처리되어야 하는 일에 필요하다. (예: Ctrl + Z) | 들어온 순서대로 처리되어야 하는 일에 필요하다. (예: 줄 서서 기다리는 경우) | 해시 충돌이 적은 상황에서 효율적이다. 따라서 데이터를 잘 분산시키는 해시 함수 선택이 중요하다. 그러나 실무에서는 해시테이블이 이미 잘 구현되었을 거라는 가정하에 개발하는 경우가 많음. 따라서 최악의 경우를 보통 상정하지 않고 사용하는 경우가 많음.|

# Array vs LinkedList

| 경우 | Array | LinkedList |
| :--- | :--- | :--- |
| **특정 원소 조회** | $O(1)$ | $O(N)$ |
| **중간에 삽입 삭제** | $O(N)$ | $O(1)$ |
| **데이터 추가** | 데이터 추가 시 모든 공간이 다 차버렸다면 새로운 메모리 공간을 할당받아야 한다. | 모든 공간이 다 찼어도 맨 뒤의 노드만 동적으로 추가하면 된다. |
| **정리** | 데이터에 접근하는 경우가 빈번하다면 **Array**를 사용하자. | 삽입과 삭제가 빈번하다면 **LinkedList**를 사용하는 것이 더 좋다. |
