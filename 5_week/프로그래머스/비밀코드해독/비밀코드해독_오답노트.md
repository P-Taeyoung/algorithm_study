## 1. 문제 분석 및 원인 파악

- **목표:** 1부터 $n$ ($n \le 30$)까지의 숫자 중 5개를 뽑는 모든 조합을 생성하고, 입력된 시도(`q`)와 비교하여 시스템 응답과 일치하는 횟수 카운트.

### 나의 초기 접근 (List 기반 비교)

```python
for password in can_password:  # password는 tuple (길이 5)
    for i, try_pass in enumerate(q):  # try_pass는 list (길이 5)
        include_nums = 0
        for idx, num in enumerate(try_pass):
            if num in password:  # 리스트(튜플) 내부 검색
                include_nums += 1
```

### 문제점

1. `itertools.combinations` 결과를 `list()`로 감싸 **메모리를 과다 사용**함.
2. `if num in password:` 부분에서 리스트/튜플은 **순차 탐색(Linear Search)** 을 하므로 불필요한 연산이 발생함.

## 2. 해결 방안: Set 자료구조 활용

### 📌 Set의 핵심 원리: 해시 테이블(Hash Table)

파이썬의 `set`은 내부적으로 **해시 테이블**로 구현되어 있습니다.

| 자료구조 | `in` 연산 방식 | 시간 복잡도 |
|---|---|---|
| 리스트(List / Tuple) | 첫 번째 원소부터 순서대로 하나씩 비교 | $O(N)$ |
| 세트(Set) | 해시 함수로 메모리 주소를 바로 계산 후 한 번에 확인 | 평균 $O(1)$ |

### 📌 개선된 로직 (Set 교집합 활용)

원소 하나씩 `in`으로 비교하는 대신, `set`의 강력한 무기인 **교집합 연산(`&`)** 을 사용합니다.  
파이썬 C엔진 단에서 매우 빠르게 두 해시 테이블을 비교해 줍니다.

```python
import itertools

def solution_optimized(n, q, ans):
    answer = 0
    # q를 미리 set 리스트로 변환 (한 번만 하면 됨)
    q_sets = [set(query) for query in q]
    nums = range(1, n + 1)

    # list() 없이 바로 generator 반복 (메모리 절약)
    for password in itertools.combinations(nums, 5):
        password_set = set(password)  # 조합을 set으로 변환

        is_match = True
        for i in range(len(q)):
            # 교집합(&)의 길이를 구하는 것이 핵심!
            # 내부 루프 없이 한 번의 연산으로 겹치는 개수 파악
            if len(password_set & q_sets[i]) != ans[i]:
                is_match = False
                break

        if is_match:
            answer += 1

    return answer
```

## 3. 기존 코드 vs 개선된 코드 비교

| 비교 항목 | 기존 코드 (List) | 개선된 코드 (Set) |
|---|---|---|
| 자료구조 | 리스트 / 튜플 | 세트(Set) 기반 |
| 탐색 방식 | 순차 탐색 | 해시 테이블 검색 |
| 탐색 시간 복잡도 | $O(N)$ (원소 개수만큼) | $O(1)$ (즉시 찾음) |
| 교집합 확인 | `for`문으로 5번 직접 비교 | `&` 연산자 1번으로 끝냄 |
| 메모리 할당 | 모든 조합을 한 번에 리스트업 | Generator로 필요할 때만 생성 |

## 💡 오늘 배운 교훈 (Takeaways)

1. **"포함 여부를 여러 번 확인해야 할 때는 무조건 `set`을 떠올리자."**
2. 모든 경우의 수를 구할 때(조합, 순열 등) `list()`로 묶지 말고, 반복문에서 직접 뽑아 쓰면 메모리가 터지지 않는다 **(SIGKILL 방지)**.
3. 파이썬의 내장 집합 연산(`&`, `|`, `-`)은 직접 `for`문을 돌리는 것보다 **비교할 수 없을 정도로 빠르다.**
